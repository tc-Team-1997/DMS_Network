## Known debt items carried forward

### SOX-1: WebAuthn Assertion Cryptographic Validation (Wave C)

webauthn_assertion_id is stored in wf_actions (Wave A Workflows v2) and aml_hit_suppressions (Wave B AML v2) but not cryptographically validated server-side. The threshold check (step_up_risk_band / step_up_amount_threshold) IS enforced — server returns 403 step_up_required when missing — but a determined attacker could forge the id field. Wave C must close this loop by proxying to POST `/py/api/v1/stepup/verify` before storing.

### SOX-2: Workflow Audit Trail Bifurcation (Wave C)

Node-side wf_actions table records reason/comment/assertion for SOX audit; Python-side workflow_steps records state machine progression. Both are written to during a workflow advance but they're parallel logs, not unified. Wave C reconciles.
